\section*{Лабораторна 2.}

\begin{enumerate}
\item Додайте декілька ендопінтів до вашої бекенд аплікації. Передбачте виклик цих нових ендпоінтів фронтенд аплікацією.
    
\item Хоча б один із ендпоінтів вашої аплікації має бути GET, і хоча б один POST.

\item Передбачте можливість отримати дані за id в api url одного із ваших ендпоінтів.

Наприклад, $api/people/:id$ ендпоінт, який можна викликати $api/people/1$ та отримати запис із id 1.

\item Додайте базу даних (SQL або no-SQL) до вашої аплікації. В базі мають бути хоча б декілька таблиць/колекцій. Додайте можливість отримання запису одного типу за id іншого. У випадку SQL бази додайте one-to-many або many-to-many зв'язок між хоча б двома таблицями.
    
Наприклад, якщо ваш проект - це розклад для студентів, додайте типи students та courses, та можливість отримати список курсів, на які записаний студент, або список студентів, записаних на курс.
    
\item Додайте бекап схеми вашої бази в репозиторій та діаграму бази в документацію репозиторію.
    
\item Додайте readme.md файл до вашого проекту з чіткими поясненнями, як запустити вашу аплікацію локально.

\item Оформіть завдання данної лабораторної як PR, та надайте посилання на нього в moodle.

\end{enumerate}
